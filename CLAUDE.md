# SmartMail Desktop 开发规则

## 1. 开局先读清现场
每次开始任务，先复述问题和修改范围，别着急动手改。确认理解后再动手。

## 2. 大任务拆小步骤
遇到大任务先列步骤，一次只做一小块。每完成一块说明改了哪些文件，方便检查和回滚。

## 3. 动手前先出方案
如果任务影响面大，先给短方案：改哪些文件、为什么改、有什么风险。确认后再动手。

## 4. 项目规则
- 不改无关文件
- 输出用中文
- 改完跑测试
- 不顺手重构
- 每阶段完成后向用户汇报，确认后才进入下一阶段

## 5. 改一轮验一轮
每改完一小轮及时验证。改页面就打开页面看，改接口就跑接口，改类型就跑 type check。

## 6. 会话乱了就收口
如果状态混乱，先总结当前状态、改了哪些文件、还有啥问题，然后开新会话继续。

## 7. 收尾要交代清楚
任务完成后列出：改了什么、为什么改、怎么验证、有啥风险。有测试没跑要说明原因。
